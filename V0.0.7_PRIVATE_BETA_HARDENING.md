# Computer Warrior v0.0.7 — Private Beta Hardening

## Purpose

Prepare the already-tested local tracker and D1 sync system for a deliberately
small, invite-only Cloudflare beta. This release does not deploy anything by
itself and does not change the local tracker privacy boundary.

## Changes

- Adds `cloudflare/wrangler.beta.jsonc`, separate from local-only
  `wrangler.jsonc`, with the remote `computer-warrior-beta` D1 binding.
- Adds D1 migration `0002_auth_rate_limits.sql` for private-beta auth attempt
  throttling.
- Requires `AUTH_PEPPER` and `INVITE_CODE` Worker secrets in beta mode.
- Requires a correct invite code for beta registration.
- Uses the server-held auth pepper for beta password and session-token hashes.
- Throttles registration and login attempts per hashed client address.
- Adds an invite-code field to the local Online dialog. It is sent only for a
  registration attempt, immediately cleared by the page, and never saved in
  `online_sync.json`.
- Keeps existing local D1 password/session behavior compatible so local
  development accounts do not silently break.

## Deliberate limits

This is an application-level guardrail for a small private beta. It is not a
substitute for Cloudflare platform controls or a production account-recovery
system. Registration remains invite-only until two-device sync, logout,
expiry, and recovery QA are complete.

## Required before deployment

1. Run `verify_project.bat` on Windows.
2. Commit and push this release.
3. Set `AUTH_PEPPER` and `INVITE_CODE` with `wrangler secret put` using
   `wrangler.beta.jsonc`. Never paste either secret into Git, a recording, or
   chat.
4. Apply remote migrations, deploy the beta Worker, and create a new beta test
   account using the invite code.
