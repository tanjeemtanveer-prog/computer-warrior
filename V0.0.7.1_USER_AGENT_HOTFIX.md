# Computer Warrior v0.0.7.1 — Cloudflare User-Agent Hotfix

## Observed issue

The local dashboard sends online-account requests through Python's standard
library. Cloudflare Browser Integrity Check returned Error 1010 before the
private beta Worker received those requests because the default
`Python-urllib/<version>` request signature was rejected.

## Fix

`computer_warrior/online.py` now sends a stable browser-compatible User-Agent
alongside the existing `Accept`, JSON content type, and optional Bearer token
headers. The header identifies no account, device, typed content, raw input
events, or XP totals.

## Verification

- Added a regression test that captures the outgoing request and verifies the
  User-Agent header.
- Existing tracker, offline queue, credential-store, Worker, migration, and
  dashboard tests remain part of `verify_project.bat`.

## Deployment impact

This is a local Python-client change only. Do not rerun remote D1 migrations or
redeploy the Worker for this hotfix.
