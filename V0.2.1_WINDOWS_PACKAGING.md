# Computer Warrior v0.2.1 — Windows Packaging Foundation

## Purpose

This release adds a repeatable Windows package build for the existing local
tracker. It produces a **one-folder** bundle with the dashboard files included.
It does not alter the tracker privacy boundary, Cloudflare Worker, D1 database,
secrets, analytics, or public landing page.

## Build safety rules

- Start from a clean Git working tree. The script refuses to package an
  uncommitted source tree and records its exact Git revision in `BUILD_INFO.txt`.
- Use a short output path such as `%USERPROFILE%\cwbuild\...`. Windows long paths
  prevented the earlier PyInstaller install inside the deeply nested source
  folder.
- The script refuses to reuse an existing build directory. Choose a new
  `-BuildRoot` for each build so an older bundle is never overwritten.
- Build the console variant first. It gives visible startup and shutdown output
  during QA. Create a `-Windowed` candidate only after that console build passes.

## Prerequisites

- Windows 10 or 11.
- Python launcher available as `py -3`.
- Internet access for the isolated virtual environment to download the pinned
  build tools on its first run.
- A clean source checkout.

## Console beta build

From the repository root, run:

```powershell
.\build_windows_beta.ps1 `
  -BuildRoot "$env:USERPROFILE\cwbuild\computer-warrior-v0.2.1-console"
```

The script creates a fresh virtual environment, installs `pynput==1.8.2`,
`PyInstaller==6.21.0`, and `pyinstaller-hooks-contrib==2026.6`, then writes:

```text
%USERPROFILE%\cwbuild\computer-warrior-v0.2.1-console\dist\ComputerWarrior\
  ComputerWarrior.exe
  BUILD_INFO.txt
  SHA256SUMS.txt
  _internal\web\index.html
```

`SHA256SUMS.txt` lists every bundled file except the manifest itself. The
full executable SHA-256 and the source commit are in `BUILD_INFO.txt`.

## Windows package QA

1. Press F10 in any source-run tracker and wait for `XP saved successfully`.
2. Copy the local state files to a dated backup folder:
   `%LOCALAPPDATA%\ComputerWarrior\stats.json`, `stats.json.bak`, and
   `online_sync.json` when present.
3. Confirm port 8765 is not already listening.
4. Run `ComputerWarrior.exe` from the console bundle.
5. Confirm the local dashboard opens at `http://127.0.0.1:8765/` and that
   Today/Lifetime XP and the existing signed-in profile are preserved.
6. Verify physical input awards only aggregate XP, F9 pauses/resumes, and F10
   saves and exits cleanly.
7. Relaunch the packaged executable. Confirm state, focus-quest history, and
   local online session restore correctly.
8. Check `SHA256SUMS.txt`, preserve the `BUILD_INFO.txt`, and record the result
   before sharing the folder with any tester.

Warnings about unused macOS/Linux `pynput` backends (`Quartz`, `Xlib`, or
`evdev`) are expected when the build collects `pynput` for its dynamic Windows
imports. They do not represent missing Windows runtime dependencies.

## Windowed candidate

After a console build passes, build to a new folder:

```powershell
.\build_windows_beta.ps1 `
  -Windowed `
  -BuildRoot "$env:USERPROFILE\cwbuild\computer-warrior-v0.2.1-windowed"
```

Repeat the same state, input, online-sync, exit, and relaunch QA. A windowed
bundle hides diagnostic console output, so it is not a substitute for the
console build.

## Not included yet

- Installer and uninstaller.
- Code signing or a guarantee that Windows SmartScreen will not warn.
- Automatic updates.
- Public download link or self-service public account registration.

Those are later release gates, after the windowed candidate is tested.
