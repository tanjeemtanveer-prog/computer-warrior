# Computer Warrior v0.2.0 — Public Site and Analytics Foundation

## Outcome

This release adds the first public-facing Computer Warrior website to the
existing Cloudflare Worker deployment. It also adds a privacy-limited product
analytics foundation without putting visitor traffic into D1.

## Public site

- Serves a responsive landing page from Cloudflare Workers Static Assets.
- Explains the product benefits, game loop, focus quests, levels, optional
  leaderboard, local-first behavior, and exact privacy boundary.
- Includes a dedicated `/privacy/` page and branded 404 page.
- Uses local CSS, JavaScript, SVG, and system fonts. It has no third-party
  images, fonts, frameworks, or runtime dependencies.
- Keeps public account actions explicitly marked as upcoming. It does not show
  a fake Join or Sign in flow while the beta still uses desktop-owned,
  invite-only authentication.
- Leaves the existing localhost dashboard in `web/index.html`; it is not copied
  into the public asset directory.

## Routing

Both Wrangler configurations now serve `cloudflare/public/` as static assets.
Only `/api/*` is forced through Worker code. This keeps public page requests
cheap while the existing health, account, device, sync, and leaderboard routes
continue using the Worker and D1.

## Visitor analytics

Daily visitors, page views, referrers, device aggregates, and performance are
designed to use Cloudflare Web Analytics. The generated Web Analytics site
token is not invented or committed by this release. The owner enables it later
by replacing the marked comments in the two public HTML pages with the exact
Cloudflare-generated snippet.

## Product analytics

The beta Wrangler configuration adds an optional Workers Analytics Engine
binding named `ANALYTICS` for dataset `computer_warrior_product`.

The Worker records only six allowlisted successful events:

- account registration;
- login;
- device registration;
- accepted non-duplicate XP sync;
- leaderboard enable; and
- leaderboard disable.

The schema accepts no free-form dimensions and contains no username, email,
account ID, device ID, label, token, IP address, user-agent, or raw activity.
If the analytics binding is missing or unavailable, the product action still
succeeds.

`cloudflare/ANALYTICS.md` defines the data boundary and Web Analytics setup.
`cloudflare/analytics-queries.sql` contains initial owner-side product queries.

## Deployment impact

- D1 migration: none.
- Worker secret change: none.
- Worker deployment: required to publish the landing page and Analytics Engine
  binding.
- Desktop data migration: none.
- Existing account and XP data: unchanged.

Before deployment, run `verify_project.bat`. After deployment, verify `/`,
`/privacy/`, `/api/health`, a signed-in sync, and the opt-in leaderboard.
