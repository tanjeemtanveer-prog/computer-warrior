# Computer Warrior v0.1.1 — Focus Quests

## Purpose

Focus Quests add an intentional work loop to Computer Warrior without claiming
to know what the user was doing. A user chooses a 25-minute, 50-minute, or
custom 5–180 minute timer. The tracker continues to count its existing four
anonymous aggregate XP categories exactly as before.

## Privacy boundary

The app stores only completed quest records containing:

- Local calendar day.
- Planned duration in minutes.
- Aggregate XP earned while the quest ran.

It does not store a start/end timestamp, key identity, typed text, application,
website, window title, cursor position, clicked location, screenshot, clipboard
content, or an input-event log. The active countdown is held only in memory and
ends if the app exits or restarts.

## Rules

- A completed quest adds **no bonus XP**.
- The XP value is the difference between lifetime aggregate XP at quest start
  and lifetime aggregate XP when its timer completes.
- A quest can be paused/resumed or abandoned manually.
- Completion is detected by the local app while it is running; the last 20
  completed records are retained locally.
- Quest records are never synchronized to the Worker, D1, or leaderboard.

## Manual Windows QA

1. Run `verify_project.bat` and confirm all Python and Worker tests pass.
2. Launch the existing local dashboard and find **Focus Quest** below Daily
   Momentum.
3. Start a 25-minute quest. Confirm the countdown appears and Start buttons are
   disabled.
4. Press **Pause quest**, wait a few seconds, and confirm the countdown stops.
   Press **Resume quest** and confirm it continues.
5. Press **Abandon**. Confirm no completion record appears.
6. Start a short custom quest only if you wish to wait for it. On completion,
   confirm one record shows its local date, planned duration, and aggregate XP.
7. Open **Online** and confirm the quest history is not shown there; use Sync
   now only for the existing aggregate XP sync.
