# Computer Warrior v0.0.7.2 — Cloudflare Free CPU Auth Hotfix

## Observed issue

The invite request reached the beta Worker after v0.0.7.1, but registration
returned the generic Worker 500 response. The Worker used 310,000 PBKDF2-SHA-
256 rounds for beta password hashes. That setting exceeded the 10 ms CPU limit
of the Cloudflare Workers Free plan during registration.

## Fix

- Sets beta PBKDF2-SHA-256 to 20,000 iterations.
- Keeps a unique random salt per password.
- Keeps `AUTH_PEPPER` server-only; it is never written to D1, the dashboard,
  Git, or local tracker state.
- Keeps the hashed-address rate limit for registration and sign-in.
- Adds a contract test that prevents an accidental return to the unsuitable
  beta work factor.

## Deployment

No D1 migration and no secret rotation are required. Deploy the changed Worker
with `npm run deploy:beta`. The old 500 attempts may have filled the beta
auth-attempt window, so wait 16 minutes after the final failed attempt before
trying Create account again.
