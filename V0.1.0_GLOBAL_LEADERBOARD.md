# Computer Warrior v0.1.0 — Global Leaderboard

## User-facing behavior

- The Online profile contains `Lifetime` and `Today (UTC)` leaderboard tabs.
- Public participation is opt-in. Accounts are private by default.
- A participant outside the top 25 sees their own rank below the public list.
- The local app continues to own the session token. The browser sees only the
  local dashboard response, never the token itself.

## Data boundary

Public leaderboard data is limited to:

- rank
- username
- accepted aggregate XP total

It excludes device ID, device label, session information, key names, typed
content, mouse positions, input history, app usage and websites.

`accepted XP` means an aggregate XP batch that the Worker accepted after its
schema, rate and sequence checks. It is not a claim of cheat-proof proof of
human work.

## Deployment

From `cloudflare` in the production checkout:

```powershell
npm run db:migrate:beta
npm run deploy:beta
```

No secret rotation is required. Existing accounts must opt in once from the
Online profile before they appear on the global board.

## Verification

1. Run `verify_project.bat`.
2. Apply migration 0003 remotely and deploy the Worker.
3. In the local dashboard, open Online and enable `Show my username and
   accepted XP publicly`.
4. Confirm the lifetime board shows the account and the `Today (UTC)` board
   shows XP accepted for the Worker UTC date.
