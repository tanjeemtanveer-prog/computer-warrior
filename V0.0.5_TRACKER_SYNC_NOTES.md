# Computer Warrior v0.0.6 — Secure local session and Git foundation

## What changed

- The local Python dashboard now has an **Online** account dialog.
- The browser communicates only with `127.0.0.1:8765`; it never receives the
  Worker session token.
- Python creates a random UUID per installation and registers it as a D1 device.
- New aggregate XP is merged into one local batch for five minutes before it
  becomes an ordered queue entry. Sync now and a clean exit seal it immediately.
- Queue entries survive an offline Worker and retry unchanged on the next sync.
- Normal background sync attempts happen at most once per five minutes and do
  not poll the leaderboard. The leaderboard refreshes when the Online panel
  opens or after Sync now.
- Fixes the Online-panel refresh endpoint so its response is always JSON rather
  than a local HTML 404 page.
- While the Online panel is open, its local pending XP, verified total and
  error state refresh every two seconds. This does not request Worker or D1.
- The local session token now lives in Windows Credential Manager, not the
  online_sync.json state file. Existing plaintext token state is migrated once
  and removed.
- Existing personal XP before first sign-in becomes the sync baseline; it is not
  retroactively uploaded to an account leaderboard.
- The dialog shows account verified XP, pending updates, last error, and the
  top lifetime leaderboard entries.

## Windows local QA

1. In the `cloudflare` folder, run the Worker with the same persistence path:

   ```powershell
   $state = Join-Path $env:LOCALAPPDATA "ComputerWarrior\wrangler-local-state"
   npm run dev -- --persist-to "$state"
   ```

2. In the project root, run `launch_web_dashboard.bat`.
3. In the browser, open **Online** and sign in as `tanveer_local` using local
   Worker URL `http://127.0.0.1:8787`.
4. Generate at least 10 seconds of normal input activity, then choose
   **Sync now**. The app will normally batch the same activity for five
   minutes before automatically uploading it.
5. In the Online dialog, confirm the pending count reaches zero and verified
   account XP changes. In a separate PowerShell terminal, confirm with:

   ```powershell
   Invoke-RestMethod http://127.0.0.1:8787/api/leaderboard?period=lifetime
   ```

6. Stop the Worker, generate more activity, then choose **Sync now**. Confirm
   the dialog shows a waiting/error state while tracking still works.
7. Restart the Worker and choose **Sync now** again. Confirm the pending count
   returns to zero and the leaderboard gains the aggregate batch once.

## Scope boundary

This is a local-only beta. It is not a deployment package, and it does not
claim complete client tamper prevention. The Worker still applies its device
sequence and XP-rate validation before accepting leaderboard XP.
