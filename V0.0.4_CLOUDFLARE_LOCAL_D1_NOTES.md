# Computer Warrior v0.0.4 — Cloudflare local D1 foundation

## Included

- Wrangler Worker source in `cloudflare/src/index.js`.
- Local D1 schema migration in `cloudflare/migrations/0001_initial_schema.sql`.
- Accounts, local-development sessions, devices, sync entries, account totals,
  daily totals, security flags, and public leaderboards.
- Idempotent device update API: a device can add only its next sequence number;
  a retry of that exact data does not add XP again.
- Per-device totals are preserved and account totals are the combined result.

## Not included yet

- No Cloudflare deployment, D1 creation, secrets, or remote database writes.
- No Python tracker sync client or browser account/leaderboard controls.
- No claim of complete anti-cheat protection; validation is a first fairness layer.

## Verification completed here

- Worker JavaScript syntax check passed.
- Four Worker contract tests passed.
- Wrangler applied migration `0001_initial_schema.sql` to local D1.
- Local D1 table query confirmed the expected schema.

## Windows manual run

1. Open PowerShell in `cloudflare`.
2. Run `setup_and_test_local.bat`.
3. Run `start_local_worker.bat`.
4. Open `http://127.0.0.1:8787/api/health`.

The Worker should report `{ "ok": true, "environment": "local", "storage": "D1" }`.
