# Computer Warrior v0.2.0 exact Windows upgrade guide

This guide upgrades the existing Git repository to the public landing-page and
analytics-foundation release. It does not migrate D1 or rotate Worker secrets.

## 1. Extract the release and define both folders

Open PowerShell in the existing Git project, then set the extracted v0.2.0
folder and existing repository paths:

```powershell
$source = "C:\Users\tanve\Downloads\Computer Hero\Computer_Warrior_v0.2.0_Public_Site_Analytics\computer_warrior_v0.2.0_public_site_analytics"

$target = "C:\Users\tanve\Downloads\Computer Hero\Computer_Warrior_v0.0.6_Security_Git_Foundation\computer_warrior_v0.0.6_security_git_foundation"

if (-not (Test-Path "$source\cloudflare\public\index.html")) {
  throw "The v0.2.0 source folder is incorrect. Stop here."
}

if (-not (Test-Path "$target\.git")) {
  throw "The target is not the existing Git project. Stop here."
}

Set-Location $target
git status
```

Continue only when Git reports a clean working tree.

## 2. Copy the release files

```powershell
$files = @(
  "README.md",
  "QA_NOTES.md",
  "SOURCE_SHA256SUMS.txt",
  "self_test.py",
  "computer_warrior\config.py",
  "cloudflare\README.md",
  "cloudflare\ANALYTICS.md",
  "cloudflare\analytics-queries.sql",
  "cloudflare\package.json",
  "cloudflare\package-lock.json",
  "cloudflare\src\index.js",
  "cloudflare\test\contract.test.js",
  "cloudflare\wrangler.jsonc",
  "cloudflare\wrangler.beta.jsonc",
  "V0.2.0_PUBLIC_SITE_ANALYTICS.md",
  "V0.2.0_UPGRADE_GUIDE.md"
)

foreach ($file in $files) {
  $destination = Join-Path $target $file
  New-Item -ItemType Directory -Force -Path (Split-Path $destination) | Out-Null
  Copy-Item -LiteralPath (Join-Path $source $file) -Destination $destination -Force
}

New-Item -ItemType Directory -Force -Path "$target\cloudflare\public" | Out-Null
Copy-Item -Path "$source\cloudflare\public\*" `
  -Destination "$target\cloudflare\public" `
  -Recurse `
  -Force
```

## 3. Install and run all automated tests

```powershell
Set-Location "$target\cloudflare"

Set-Location $target
.\verify_project.bat
```

Expected results:

- Python: 41 tests pass.
- Worker/site: 14 tests pass.
- No test fails.

The line-ending warnings on Windows are informational. Stop if any test fails.

## 4. Preview the public website locally

The existing D1 local state can be reused:

```powershell
Set-Location "$target\cloudflare"
$state = Join-Path $env:LOCALAPPDATA "ComputerWarrior\wrangler-local-state"
npm run dev -- --persist-to "$state"
```

Keep that terminal open. Visit:

- `http://127.0.0.1:8787/`
- `http://127.0.0.1:8787/privacy/`
- `http://127.0.0.1:8787/api/health`
- `http://127.0.0.1:8787/a-page-that-does-not-exist`

Verify:

1. The landing page, not API JSON, appears at `/`.
2. Desktop and phone-width layouts have no horizontal scrolling.
3. The phone navigation opens, closes, and responds to Escape.
4. Every landing-page link works.
5. `/privacy/` lists the exact local and online data boundaries.
6. The unknown path shows the branded 404 page with HTTP status 404.
7. `/api/health` still returns `ok`, `environment`, and `storage`.
8. No Join, Sign in, or newsletter form falsely claims to be available yet.

Press `x` in the Wrangler terminal when preview QA is complete.

## 5. Refresh checksums after tests

`verify_project.bat` rewrites `QA_NOTES.md`, so regenerate the source manifest
after the tests:

```powershell
Set-Location $target
$root = (Get-Location).Path
$excluded = '\\(\.git|node_modules|\.wrangler|__pycache__|\.pytest_cache|\.venv|venv)(\\|$)'

$lines = Get-ChildItem -Recurse -File | Where-Object {
  $_.Name -ne "SOURCE_SHA256SUMS.txt" -and
  $_.Name -notin @("online_sync.json", "stats.json", "stats.json.bak", ".dev.vars", ".env") -and
  $_.Extension -notin @(".pyc", ".zip") -and
  $_.FullName -notmatch $excluded
} | Sort-Object FullName | ForEach-Object {
  $hash = (Get-FileHash -Algorithm SHA256 -LiteralPath $_.FullName).Hash.ToLowerInvariant()
  $relative = "./" + $_.FullName.Substring($root.Length + 1).Replace("\", "/")
  "$hash  $relative"
}

$lines | Set-Content -Encoding ascii "SOURCE_SHA256SUMS.txt"
```

## 7. Deploy the reviewed beta

No D1 migration is required.

```powershell
Set-Location "$target\cloudflare"
npx wrangler secret list --config wrangler.beta.jsonc
npm run deploy:beta
```

Confirm that `AUTH_PEPPER` and `INVITE_CODE` are still listed before deploying.
Do not recreate or rotate them.

After deployment:

```powershell
$beta = "https://computer-warrior-beta.tanjeem-solitaire.workers.dev"

Invoke-WebRequest "$beta/" -UseBasicParsing |
  Select-Object StatusCode, ContentType

Invoke-RestMethod "$beta/api/health"

Invoke-WebRequest "$beta/privacy/" -UseBasicParsing |
  Select-Object StatusCode, ContentType
```

Expected:

- landing page: status 200, HTML;
- health: `ok = True`, `environment = beta`, `storage = D1`;
- privacy page: status 200, HTML.

Then launch the desktop app, sign in, complete one manual sync, and confirm the
existing leaderboard still loads. The first successful event creates the
Analytics Engine dataset automatically.

## 8. Enable daily visitor analytics separately

Do this only after the public pages pass the deployed smoke test:

1. Open Cloudflare Dashboard.
2. Go to **Analytics & Logs → Web Analytics**.
3. Add the public Computer Warrior site.
4. Copy the exact JavaScript beacon snippet Cloudflare generates.
5. Replace the `CLOUDFLARE_WEB_ANALYTICS` comment in:
   - `cloudflare\public\index.html`
   - `cloudflare\public\privacy\index.html`
6. Rerun `.\verify_project.bat`.
7. Commit and deploy that small change.
8. Visit both pages in a private browser window and confirm page views appear
   in Web Analytics.

Do not add visitor writes to D1. Read `cloudflare\ANALYTICS.md` before extending
the event allowlist or adding newsletter/account data.
