# v0.0.6 Windows QA

## Credential Manager migration

1. Stop every earlier Computer Warrior instance.
2. Start the local Worker with the same persistent state directory.
3. Start this version of Computer Warrior.
4. Open Online. A previous valid local sign-in should remain signed in.
5. Open the file at %LOCALAPPDATA%\ComputerWarrior\online_sync.json in a text
   editor. Confirm it contains credential_target and does not contain
   session_token.
6. Open Windows Credential Manager, then Windows Credentials. Under Generic
   Credentials, confirm there is an entry whose name starts with
   ComputerWarrior.Session.
7. Use Sync now and confirm pending XP reaches zero.
8. Choose Sign out. Reopen Online and confirm it requires sign-in again.

If the older token cannot be migrated securely, the expected behavior is a safe
sign-out with a request to sign in again. Do not copy the credential entry or
online_sync.json to another PC.

## Git foundation

1. Run verify_project.bat and confirm both test suites pass.
2. Run init_git_repository.bat once.
3. Run git status and confirm node_modules, .wrangler, online_sync.json and
   ZIP files are absent from the list.
4. Follow GIT_WORKFLOW.md for the first commit.
